# DAY5-网站上线不是把电脑一直开着

## 内部今日目标

亲手验证并理解“本地项目 → GitHub → Vercel → 公网网站”的上线链路。

## 概念及相关类比

- **本地项目**：像自己家的书房，书稿和修改都先放在自己的电脑里。
- **Git**：像给每次修改拍一张存档照片，可以确认改了什么，也可以回看历史。
- **GitHub**：像云端仓库，保存已经推送上去的代码版本。
- **Vercel**：像自动印刷和发布平台，从 GitHub 获取新版本并部署网站。
- **公网网址**：像公众都可以访问的书店，不需要进入我的电脑，也不需要我的电脑一直开着。

## 一个小动作

我在本地开发环境没有运行的情况下，刷新了 VoiceMore 学习网站。网站仍然可以访问，这说明页面来自 Vercel 的服务器，而不是我的电脑。

我也完成了一个判断：只修改本地文件，公网网站不会立刻变化，因为新版本还没有推送到 GitHub，Vercel 也就没有收到新的部署内容。

## 一张知识卡

**是什么：** 网站上线，是把本地项目通过 GitHub 交给 Vercel，部署成公众可以访问的网站。

**什么时候用：** 当我修改了网站内容，希望其他人通过公网网址看到新版本时。

**怎么做：** 在本地修改并检查内容，用 Git 保存版本，推送到 GitHub，等待 Vercel 自动部署，最后打开公网网址验证结果。

```text
本地项目 → Git 提交 → 推送 GitHub → Vercel 自动部署 → 公网验证
```

