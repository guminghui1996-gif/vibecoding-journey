# DAY4-Git 是 AI 修改前后的安全检查点

## 内部今日目标

学会在 AI 修改前后，用 Git 确认现场是否干净、改了什么、有没有保存成版本点。

## 概念及相关类比

- **工作区干净**：像开工前桌面收拾好了。技术上是没有未提交或未跟踪改动。
- **分支**：像在单独草稿本里做实验。技术上是一个独立的开发线。
- **提交**：像拍一张存档照片。技术上是把一组改动保存成版本点。
- **diff**：像对比修改前后的两张图。技术上是查看具体改了哪些行。

## 一个小动作

我亲自运行 Git 命令，确认当前分支、最近存档点，以及某次提交实际修改了哪些文件。

## 一张知识卡

**是什么：** Git 是 AI 修改前后的安全检查点。

**什么时候用：** 每次让 AI 改代码前、改完后、提交前、发现问题想回看历史时。

**怎么做：** 用 `git status` 看现场，用 `git diff` 看改动，用 `git log` 看历史，用 `git show --stat` 看某次提交改了哪些文件。

```powershell
git status --short --branch
git log --oneline --decorate -5
git show --stat --oneline 80e9d49
```
