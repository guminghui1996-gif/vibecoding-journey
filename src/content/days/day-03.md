# DAY3-先看每一棒交接

## 内部今日目标

不用逐行读懂全部代码，先看一段语音如何一步步变成可插入的文字。

## 概念及相关类比

- **数据流**：像快递从寄件人到分拣站再到收件人。技术上是信息在模块之间传递。
- **输入**：像快递从哪里来。技术上是一个模块收到什么。
- **输出**：像快递交给下一站什么。技术上是一个模块产出什么。
- **边界**：像每个岗位只负责自己的环节。技术上是一个模块负责什么、不负责什么。

## 一个小动作

我亲自判断：以后改 DeepSeek，优先看“文本处理”这一棒；以后改本地 Whisper，优先看“转写”这一棒。

## 一张知识卡

**是什么：** 数据流是看清一个产品里信息如何从输入一步步变成输出。

**什么时候用：** 当我看不懂代码、想替换服务商、想定位 Bug 或想让 AI 修改某一段流程时。

**怎么做：** 先画出输入、处理、输出，再问每一棒负责什么、不负责什么。

```text
Alt -> 目标窗口 -> 录音 -> WAV -> 转写 -> 文本处理 -> 插入 -> 结果窗口兜底
```

## 一个可复用 Prompt

```text
请不要逐行解释代码。请先帮我画出这个功能的数据流：输入是什么、经过哪些模块、每一步输出什么、哪里最可能需要修改。
```
